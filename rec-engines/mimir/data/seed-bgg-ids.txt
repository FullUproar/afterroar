# Mimir seed BGG IDs -- onboarding cold-start pool
#
# Format: one ID per line, optionally followed by # comment.
# Empty lines and lines starting with # are ignored by fetch-bgg.mjs
# (it splits on whitespace and filters out non-numeric tokens).
#
# Hand-curated for diversity across:
#   - Weight tiers (party, light, medium-light, medium, medium-heavy, heavy)
#   - Mechanic families (engine, deck-builder, area control, worker placement,
#     deduction, dexterity, social deduction, cooperative, wargame, abstract,
#     drafting, push-your-luck, racing, narrative)
#   - Theme breadth (fantasy, sci-fi, civ, abstract, train, nature, food,
#     historical, horror, espionage)
#   - Era (classics + modern hits)
#
# Goal: ~60 games such that any new player presented with this pool can
# pick 5 they love + 2 they can't stand and produce a useful seed taste
# vector (per design doc § 8).
#
# Refresh quarterly based on what is actually being played in the FLGS
# network. This is v0 hand-curated by Claude during Sprint 1.0.1 -- a
# human pass to validate IDs and adjust based on real-world usage data
# is on the open-questions list (mimir/README.md).
#
# Notes on IDs:
# - BGG IDs are stable but games occasionally get renumbered when entries
#   merge. If `fetch-bgg.mjs` returns no item for a given ID, check BGG
#   directly and update this file.
# - Where multiple editions exist, prefer the most-played edition's ID
#   (e.g., War of the Ring 2nd Edition over the original).
# - Some choices are deliberately classical (Catan, Carcassonne) even
#   when newer games might dominate hot lists -- onboarding needs games
#   most respondents have an opinion on, not just current darlings.
